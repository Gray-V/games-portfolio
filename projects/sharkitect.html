Sharkitect Game Engine

A custom physics-based game engine built from scratch in C, featuring real-time collision detection and advanced vector mathematics

Project Overview

Sharkitect is a comprehensive physics-based game engine developed as an independent study project by Grayson von Goetz und Schwanenfliess and Matthew Lee. Written entirely in C, this engine demonstrates advanced understanding of low-level programming principles, computational physics, and real-time graphics rendering. The project showcases sophisticated collision detection algorithms and vector mathematics implementations that power interactive physics simulations.

Technical Architecture

  • Physics Engine Core - Custom-built physics simulation system capable of real-time collision detection, response calculations, and dynamic object interactions
  • Low-Level C Programming - Entire engine developed in C, demonstrating mastery of memory management, pointer arithmetic, and system-level programming concepts
  • Vector Mathematics Library - Comprehensive mathematical foundation utilizing custom vector operations for accurate physics calculations and transformations
  • OpenGL Integration - Advanced graphics rendering pipeline using OpenGL for hardware-accelerated 2D and 3D graphics with optimized performance
  • Real-Time Simulation - Efficient algorithms ensuring smooth, responsive physics interactions at consistent frame rates

Development Process & Research

The development of Sharkitect involved extensive research into game engine architecture, computational physics, and graphics programming. The project required deep understanding of fundamental computer science concepts including data structures, algorithms, and mathematical modeling.
